## Required Page Structure - STRICTLY FOLLOW THIS FORMAT

Each tier page MUST include the following sections in this order:

### Section 1: Executive Summary (200-300 words)

**What it is:**
- Clear, jargon-free explanation a GP can understand in 60 seconds
- One-sentence "elevator pitch" for the tier
- 3-5 bullet points of key value drivers

**Financial snapshot:**
```
Investment: $X.XM (core) + $XXK per company (customization)
Annual Return: $XX-XXM (Year 3 steady state)
ROI: XXX%
Payback: X-XX months

```

### Section 2: Problem Statement (300-400 words)

**Current State Across Portfolio:**
- What manual process exists today?
- Why is it painful? (time, cost, errors, risk)
- Quantify the pain (hours per week, cost per transaction, error rates)
- Why hasn't this been solved already?

**Portfolio-Specific Context:**
- Which companies have this problem worst?
- Variations in how companies currently handle this
- Strategic importance (is this core to value creation?)

**Technical Approach:**
- What AI technologies are used? (LLMs, ML models, computer vision, etc.)
- How does the system work? (step-by-step workflow)

- Core platform (built once) - what's included?

- Technology stack (be specific: Claude API, AWS SageMaker, etc.)

- Data flow diagram
